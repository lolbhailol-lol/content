# CrwdCtrl — Go-To-Market Strategy

> Campus by campus, community by community: the playbook for becoming India's youth experience platform.

---

## Executive Summary

CrwdCtrl's go-to-market strategy is built on a simple insight: **college campuses are concentrated, high-density, repeatable markets**. Win one campus, learn the playbook, replicate. The same applies to communities — win one run club, learn what works, scale to the next.

This document outlines the GTM approach: starting with a single campus (MIT-WPU), expanding city by city, and building network effects that compound.

## GTM Thesis

### The Wedge: College Campuses

**Why campuses?**
- Concentrated audience (thousands of students in one location)
- High event density (fests, clubs, competitions every week)
- Word-of-mouth spreads fast (everyone knows everyone)
- Repeatable playbook (every campus has the same structures)
- Low CAC (organic discovery within campus)

**The campus flywheel:**
```
Onboard 1 organizer (fest/club)
        ↓
Students register for event
        ↓
Students discover CrwdCtrl
        ↓
Students attend more events
        ↓
More organizers see value
        ↓
More organizers list events
        ↓
Campus becomes "CrwdCtrl-powered"
```

### Expand to Communities

Once campus base is established:
- Run clubs (Pune, then Mumbai)
- Trek communities (Western Ghats focus)
- Theatre collectives (city-specific)

Same playbook: win one community, learn, replicate.

## Recommendations

### Phase 1: Campus Dominance (0–90 Days)

**Goal**: Own MIT-WPU completely; prove the model.

**Target Metrics**:
- 20+ organizers (fests, clubs) listed
- 500+ registrations processed
- 80% of major campus events on CrwdCtrl

**Tactics**:

1. **Founder-led organizer sales**
   - Meet every fest committee, club president
   - Attend campus events; document for content
   - Offer hands-on onboarding support

2. **Campus ambassador program (pilot)**
   - Recruit 3–5 student ambassadors at MIT-WPU
   - Incentive: free event tickets, recognition, small cash
   - Task: promote CrwdCtrl, onboard organizers, generate content

3. **Event presence**
   - Attend every major campus event
   - Document with photos/video (content engine)
   - Run CrwdCtrl booth if possible

4. **Product improvements for organizers**
   - Reduce friction in listing process
   - Faster registration dashboards
   - Google Sheets export for legacy workflows

### Phase 2: City Expansion (90–180 Days)

**Goal**: Expand to 3–5 campuses; 2–3 community verticals in Pune.

**Target Metrics**:
- 75+ organizers across campuses
- 2,000+ registrations
- 10+ active run clubs/trek communities

**Tactics**:

1. **Campus replication playbook**
   - Document what worked at MIT-WPU
   - Identify next 3 campuses in Pune (COEP, Symbiosis, VIT Pune)
   - Recruit ambassadors at each

2. **Community vertical focus**
   - Run clubs: Partner with 5–10 Pune run clubs
   - Trek communities: Onboard Western Ghats trek groups
   - Positioning: "Find your running crew" / "Your trek starts here"

3. **Content-driven acquisition**
   - FOMO content: "What happened at [event] this weekend"
   - Discovery content: "Run clubs in Pune you didn't know about"
   - Founder content: Monthly scorecard, build-in-public

4. **Android app launch**
   - Complete Play Store submission
   - Campus-specific launch campaigns
   - Push notifications for engagement

### Phase 3: Regional Scale (180+ Days)

**Goal**: Expand beyond Pune; establish regional presence.

**Target Metrics**:
- 10+ cities/campuses
- 100+ organizers
- 5,000+ monthly registrations
- Android app: 1,000+ installs

**Tactics**:

1. **City-by-city expansion**
   - Priority cities: Mumbai, Bangalore, Delhi
   - Same playbook: campus first, then communities

2. **Self-serve organizer onboarding**
   - Reduce founder dependency
   - Automated approval workflow
   - Organizer success documentation

3. **Referral programs**
   - User referrals: "Invite a friend, get [reward]"
   - Organizer referrals: "Bring an organizer, get [reward]"
   - Community referrals: Members grow their community

4. **Strategic partnerships**
   - University associations
   - Running/fitness brands
   - Trek equipment companies

## Competitive Response

### If BookMyShow enters:

- They have distribution but not community DNA
- Focus on long-tail (campus events, local communities) they won't prioritize
- Build community moat before they notice

### If Instagram improves event features:

- Instagram is attention, not transaction
- CrwdCtrl owns the full flow (discovery → registration → payment → check-in)
- Stay close to organizer needs

### If new startup copies:

- Execution and brand win
- Speed: move fast, dominate campuses
- Community lock-in: hard to switch once history is built
